# SUG01 — 건설 목표 예약과 자동화

## 제안 요약

현재의 “건설력을 모으면 즉시 건설” 규칙은 유지하면서, 플레이어가 다음에 지을 건물을 미리 예약하는 건설 목표 목록을 추가한다. 건설 시간이 새로 생기는 것이 아니라, 조건이 충족되는 순간 예약한 건물을 자동 건설하는 편의·전략 시스템이다.

## 현재 문제

- 건설력이 충분해질 때마다 건물 패널을 열고 같은 버튼을 반복해서 눌러야 한다.
- 장시간 플레이에서는 “다음 병목을 해결하기 위한 준비”보다 버튼 확인 빈도가 플레이를 지배할 수 있다.
- 즉시 건설 규칙을 건설 대기시간 방식으로 바꾸면 현재 설계와 충돌하고 idle 흐름도 느려질 수 있다.

## 권장 구현

1. 건물별로 `즉시 건설`과 `목표 추가` 액션을 제공한다.
2. 목표는 `건물 ID`, `목표 수량`, `반복 여부`, `우선순위`만 저장한다.
3. 건설력·인구·기술 조건이 모두 충족되면 기존 `TryBuild()`를 호출한다.
4. 조건이 부족하면 예약은 유지하고 부족 사유를 표시한다.
5. 자동 건설은 새 계산 규칙을 만들지 않고 수동 건설과 동일한 검증·비용 차감 경로를 사용한다.

## 플레이 시나리오

플레이어가 철 생산을 준비하며 `철 광산 3개 → 공구공장 2개 → 건설부문 1개`를 예약한다. 건설력이 쌓일 때마다 조건을 만족하는 첫 목표가 즉시 건설된다. 철 광산 해금 전에는 해당 목표가 대기하고, UI는 `기술 필요`를 표시한다. 플레이어는 경제가 성장하는 동안 다음 생산 체인을 설계하는 데 집중할 수 있다.

## 기대 효과와 위험

- 반복 클릭을 줄이면서 기존 즉시 건설 규칙을 보존한다.
- 건설 순서 자체가 전략 선택이 된다.
- 자동화가 너무 일찍 열리면 수동 조작의 의미가 줄 수 있으므로 고대 또는 고전 기술 보상으로 단계적으로 개방하는 방안을 권장한다.

## 후속 결정 필요

- 목표 목록 최대 길이
- 반복 목표의 종료 조건
- 조건 불충족 목표를 건너뛸지, 전체 목록을 정지할지
- 자동화 해금 기술과 시대
