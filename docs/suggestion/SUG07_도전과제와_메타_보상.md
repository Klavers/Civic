# SUG07 — 도전과제와 메타 보상

> 상태: 제안
> 연계: SUG06 환생, SUG08 시작 문명, SUG09 국가 설립, SUG10 정치·사회 체계, SUG11 이벤트, SUG12 불가사의, SUG13 위인

## 1. 제안 목적

일반 idle 게임의 성장 이정표와 Paradox Interactive 게임처럼 특이한 조건 조합을 모두 지원하는 도전과제 시스템을 도입한다. 도전과제는 단순 수집 목록이 아니라 새로운 플레이 방식의 안내서로 사용한다.

보상은 세 층으로 분리한다.

- 기본 보상: 환생 포인트, 프로필 배지, 연대기 기록
- 선택 보상: 특정 시작 문명·이벤트·국가 설립 조건 해금
- 제한적 영구 보상: 작은 영구 모디파이어 또는 유산 구매권

영구 모디파이어는 모든 도전과제에 주지 않고 고난도·고유 도전과제에만 부여한다. 단순 누적 업적에 강한 생산 배율을 주면 플레이 순서가 사실상 강제되기 때문이다.

## 2. 도전과제 공통 규칙

- `계정 도전과제`: 한 번 달성하면 영구 유지한다.
- `런 도전과제`: 환생 전 현재 런에서만 판정하며 완료 시 계정 기록으로 승격한다.
- `연속 조건`: 특정 상태를 N초 유지해야 하며 일시적인 재고 이동으로 달성할 수 없게 한다.
- `금지 조건`: 특정 건물·기술·정책을 사용하지 않았는지 런 플래그로 추적한다.
- `비밀 조건`: 달성 전 제목과 힌트만 보여주고 정확한 수치는 숨길 수 있다.
- 모든 판정은 저장 가능한 achievement progress에서 수행하고 UI 텍스트 파싱에 의존하지 않는다.

## 3. 대분류 1 — 성장과 누적 이정표

전형적인 idle 성장 목표다. 초보 플레이어에게 다음 시스템을 안내하고 장기 숫자 성장을 가시화한다.

| 예시 | 조건 | 보상 | 플레이 시나리오 |
|-|-|-|-|
| 1. 번성하는 정착지 | 인구 100 달성 | 환생 포인트 3, 인구 배지 | 집과 소비재 생산을 균형 있게 늘리며 인구 시스템을 학습한다. |
| 2. 산업의 심장 | 런 최고 GDP 1,000,000 달성 | 환생 포인트 5, GDP 기록 프레임 | 고가 가공품과 품귀 가격을 활용해 생산 체인을 확장한다. |
| 3. 시대의 증인 | 미래시대 진입 | 환생 포인트 10, 다음 런 시작 과학 +3 영구 모디파이어 | 모든 시대를 한 번 경험한 장기 런에 명확한 완료 보상을 준다. |

장점은 목표가 이해하기 쉽고 진행률 표시가 명확하다는 점이다. 단점은 정상 플레이만 해도 자동 달성되어 재플레이 가치는 낮다. 환생 포인트는 주되 강한 영구 모디파이어는 마지막 단계에만 둔다.

## 4. 대분류 2 — 제약과 숙련 도전

일반 성장 규칙에 제한을 걸어 생산 체인 이해도를 시험한다.

| 예시 | 조건 | 보상 | 플레이 시나리오 |
|-|-|-|-|
| 1. 균형 예산 | 고대 진입부터 산업 진입까지 국고가 0이 되지 않음 | 환생 포인트 8, 건설부문 운영비 -1% | 건설부문을 무작정 늘리지 않고 GDP와 세율을 먼저 확보한다. |
| 2. 품귀 없는 도시 | 인구 250 상태에서 해금 자원 공급률 90% 이상을 120초 유지 | 환생 포인트 10, 품귀 경고 테두리 | 생산량보다 공급망 균형을 목표로 건물 비율을 조정한다. |
| 3. 작은 정부 | 건설부문 5개 이하로 르네상스 진입 | 환생 포인트 7, 시작 건설력 +1 | 낮은 건설력으로 우선순위를 정하고 건설 자동화 없이 효율적인 순서를 찾는다. |

장점은 시스템 숙련을 다른 방식으로 검증한다는 점이다. 단점은 실패 시 긴 런이 무효화될 수 있다. 금지 조건이 깨지기 직전에 경고하고, 이미 실패한 도전은 런 중 숨기지 말고 실패 사유를 보여줘야 한다.

## 5. 대분류 3 — 특이한 대체역사형 도전

Paradox식 업적처럼 이름과 조건이 이야기를 만들도록 설계한다. 최적 성장과 다른 비정상적 경제를 요구한다.

| 예시 | 조건 | 보상 | 플레이 시나리오 |
|-|-|-|-|
| 1. 밀로 세운 제국 | 산업시대에 밀 GDP 비중 60% 이상, 총 GDP 50,000 이상 | 환생 포인트 12, 밀 아이콘 장식 | 고급 산업 대신 농업 생산과 가격 수요를 의도적으로 유지한다. |
| 2. 돌도끼와 데이터센터 | 철 광산을 한 번도 건설하지 않고 정보시대 진입 | 환생 포인트 15, 공구 생산 +1% 영구 모디파이어 | 철 기반 기술 효과를 포기하고 석재·구리·플라스틱 중심 경로를 찾는다. |
| 3. 마지막 오두막의 공화국 | 집 1개만 보유한 상태로 공화정 수립 및 인구 150 달성 | 환생 포인트 15, 공화정 이벤트 해금 | 소비 보너스와 시작 문명 특성을 조합해 제한된 주거로 인구를 유지한다. |

장점은 커뮤니티에서 공유할 만한 독특한 서사를 만든다는 점이다. 단점은 조건이 불명확하면 공략 의존도가 높아진다. 제목·힌트·실패 조건을 명확히 제공하고 불가능 조합은 데이터 검증으로 막아야 한다.

## 6. 대분류 4 — 시스템 조합과 통치 도전

정치, 국가, 이벤트, 불가사의, 위인 등 여러 제안 시스템을 함께 사용하게 한다.

| 예시 | 조건 | 보상 | 플레이 시나리오 |
|-|-|-|-|
| 1. 개혁의 시대 | 한 런에서 정부·유권자·교육·복지 체계를 각각 1회 이상 변경 | 환생 포인트 10, 정책 변경 비용 -1% | 국고와 정당성 손실을 관리하며 연속 개혁을 수행한다. |
| 2. 세 개의 왕관 | 한 런에서 국가 설립 3단계를 연속 완료하고 최종 국가 형성 | 환생 포인트 12, 시작 문명 1개 해금 | SUG08 시작 문명에서 SUG09의 후계 국가 경로를 완주한다. |
| 3. 황금기의 설계자 | 불가사의 2개 완공, 위인 3명 은퇴, 생활수준 번영 유지 | 환생 포인트 15, 위인 후보 1회 무료 갱신 | 경제·인물·건축 시스템을 동시에 운영하는 고난도 종합 목표다. |

장점은 각 시스템이 고립되지 않고 하나의 런 목표로 결합된다는 점이다. 단점은 아직 구현되지 않은 시스템에 의존하므로 도전과제는 dependency ID를 가져야 하며 미구현 콘텐츠가 있으면 목록에서 비활성화해야 한다.

## 7. 대분류 5 — 메타·비밀·연속 도전

여러 환생과 숨은 조건을 사용해 장기 수집 동기를 제공한다.

| 예시 | 조건 | 보상 | 플레이 시나리오 |
|-|-|-|-|
| 1. 다섯 번의 새벽 | 서로 다른 시작 문명 5개로 각각 고전시대 도달 | 환생 포인트 10, 시작 문명 선택 슬롯 장식 | 같은 초반을 문명 특성별로 다르게 최적화한다. |
| 2. 역사가의 서가 | 이벤트 도감 100개 중 60개 발견 | 환생 포인트 12, 희귀 이벤트 가중치 +2% | 조건부 이벤트를 찾기 위해 정책과 자원 상태를 다양하게 바꾼다. |
| 3. 이름 없는 건국자 | 비밀 국가 3개 설립 | 환생 포인트 20, 환생 포인트 획득 +2% 영구 모디파이어 | 숨은 기술·시대·정책 조합을 추적해 장기 목표를 달성한다. |

장점은 여러 런을 연결하고 발견의 재미를 만든다는 점이다. 단점은 확률 이벤트가 포함되면 운에 의해 진행이 막힐 수 있다. 비밀 도전도 누적 보정과 확정 발생 장치를 제공해야 한다.

## 8. 보상 설계

### 권장 보상 비율

- 60%: 환생 포인트와 장식 보상
- 25%: 시작 문명·이벤트·국가 설립 경로 해금
- 15%: 작은 영구 모디파이어

영구 모디파이어 예시는 시작 과학 +3, 정책 변경 비용 -1%, 첫 불가사의 건설비 -5%처럼 상한이 분명한 값으로 제한한다. `전체 생산 +10%` 같은 범용 배율은 도전과제 필수화를 유발하므로 비권장한다.

도전과제 보상은 달성 즉시 지급하되, 환생 포인트는 현재 런 포인트와 메타 포인트를 구분해 중복 지급을 방지한다. 같은 도전과제의 반복 달성은 기록 시간·최고 점수만 갱신하고 기본 보상을 다시 지급하지 않는다.

## 9. UI 제안

- `전체 / 성장 / 제약 / 특이 / 조합 / 메타` 탭을 제공한다.
- 각 항목에 제목, 설명, 진행률, 현재 런 가능 여부, 실패 조건, 보상을 표시한다.
- 최대 3개를 추적 목록에 고정하고 HUD 알림에서 진행 변화를 보여준다.
- 비밀 도전과제는 실루엣·힌트·발견 경로만 표시하며, 한 번 조건 일부를 충족하면 상세 힌트를 추가한다.
- 런 종료 화면에서 이번 런 달성, 실패, 거의 달성한 항목을 분리해 다음 런 목표를 제안한다.

## 10. 데이터 모델 제안

| 데이터 | 주요 필드 | 역할 |
|-|-|-|
| `achievements.csv` | id, category, titleKo, descriptionKo, visibility, scope, rewardIds | 도전 정의 |
| `achievement_conditions.csv` | achievementId, conditionType, targetId, comparator, value, duration, forbidden | 조건 목록 |
| `achievement_rewards.csv` | id, rewardType, targetId, amount, maxGrant | 보상 정의 |
| `AchievementProgress` | currentValue, durationHeld, failedFlags, completedAt, runId | 런·메타 진행 |
| `AchievementSnapshot` | state, progressText, failureReason, claimableRewards | UI read model |

복합 조건은 코드에 도전과제별 if 문을 추가하지 않고 conditionType 평가기로 구성한다. 조건 판정은 이벤트 기반 누적과 1초 tick 검사를 혼합하되, 동일 이벤트 중복 처리 방지를 위해 runId와 event sequence를 저장한다.

## 11. 다른 제안과의 연계

- SUG06: 최초 완료 보상과 고난도 도전 배율을 환생 포인트에 연결한다.
- SUG08: 특정 문명 전용 도전과 모든 문명 순회 도전을 만든다.
- SUG09: 국가 설립 자체와 특수 설립 경로를 도전 조건으로 사용한다.
- SUG10: 정치 체제 조합과 개혁 횟수를 조건으로 사용한다.
- SUG11: 도전과제 완료로 이벤트 체인을 해금하거나 이벤트 선택 결과를 조건으로 사용한다.
- SUG12: 불가사의 완공 수, 제한 시간, 특정 문명 조합을 조건으로 사용한다.
- SUG13: 위인 모집·은퇴·업적 조합을 조건으로 사용한다.

## 12. 추가 결정 필요

- 일반·고난도·비밀 도전과제의 보상 상한
- 실패 조건 경고 시점과 도전 재시작 방식
- 확률 이벤트가 포함된 업적의 운 보정 규칙
- 플랫폼 업적과 게임 내부 업적의 동기화 여부
- 출시 시 제공할 총 도전과제 수와 업데이트 정책
