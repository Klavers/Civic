# SUG12 — 불가사의와 거대 건축

> 상태: 제안
> 연계: SUG06 환생, SUG07 도전과제, SUG08 시작 문명, SUG09 국가 설립, SUG10 정치·사회 체계, SUG11 이벤트, SUG13 위인

## 1. 제안 목적

일반 건물보다 훨씬 많은 건설력과 여러 자원을 요구하지만 런의 전략을 바꾸는 강력한 효과를 제공하는 불가사의를 도입한다. 불가사의는 즉시 건설하지 않고 자원 납품과 건설 진행도를 갖는 장기 프로젝트로 처리한다.

아래 비용은 첫 시뮬레이션용 제안값이며 최종 밸런스 정본이 아니다. 시대별 경제 규모 테스트 후 조정해야 한다.

## 2. 건설 공통 규칙

1. 시대·기술·문명·국가·정책 조건으로 설계도를 해금한다.
2. 플레이어가 불가사의 프로젝트를 시작하고 국고 착수비를 지불한다.
3. 필요한 자원은 재고에서 매초 일정량씩 납품한다. 부족하면 진행은 정지하지만 납품분은 보존한다.
4. 건설력은 일반 건물 건설과 경쟁하는 별도 예약량으로 소비한다.
5. 25/50/75% 단계마다 SUG11 건설 이벤트가 발생할 수 있다.
6. 완공 시 런 modifier, 도전과제, 국가·이벤트 해금을 적용한다.

기본적으로 한 번에 불가사의 1개만 건설한다. 산업시대 이후 정책·위인·유산으로 2개까지 병행할 수 있다. 한 런의 완공 불가사의 수에는 강제 상한을 두지 않되, 두 번째부터 국고 유지비가 증가하도록 한다.

## 8. 불가사의 제한과 실패 규칙

- 같은 불가사의는 한 런에 1회만 건설한다.
- 프로젝트 취소 시 미납 자원은 유지되고 납품 자원의 70%를 반환한다.
- 국고가 0이어도 진행은 정지할 뿐 납품 자원이 사라지지 않는다.
- 환생 시 미완공 프로젝트는 초기화하고 완공 기록만 메타 도감에 남긴다.
- 불가사의 효과는 sourceType=`wonder`로 modifier pipeline에 들어가며 같은 효과군 상한을 적용한다.
- 파괴 시스템은 초기 범위에서 제외한다. 사건으로 `일시 비활성`은 가능하다.

## 9. UI 제안

- 별도 `불가사의` 패널에 해금, 건설 가능, 건설 중, 완공, 미발견 탭을 둔다.
- 카드에 시대, 효과, 총비용, 현재 납품량, 예상 완공 시간, 유지비, 연계 콘텐츠를 표시한다.
- 건설 화면에서 자원별 납품 속도와 부족 원인을 표시하고 납품 우선순위를 조절한다.
- HUD에는 진행 중 프로젝트 1개와 가장 부족한 자원만 요약한다.
- 완공한 불가사의는 연대기와 3D/2D 아트 슬롯에 기록하되, 초기 구현은 prefab 카드와 일러스트로 처리한다.
- 사용자 편집 가능한 Wonder Card prefab variant와 아이콘 address를 데이터에 둔다.

## 10. 플레이 경험 장단점

### 장점

- 과잉 재고와 높은 건설력의 장기 사용처가 생긴다.
- 런마다 하나의 큰 목표와 완공 성취감을 제공한다.
- 문명·국가·정책·위인·도전과제를 결합하는 중심 콘텐츠가 된다.

### 단점 및 완화

- 비용을 잘못 잡으면 기다리기만 하는 구간이 된다: 납품 중간 이벤트와 단계 보상을 제공한다.
- 강한 효과가 필수 빌드를 만들 수 있다: 상호 배타적 장점과 유지비를 둔다.
- 자원 예약이 기존 소비를 붕괴시킬 수 있다: 납품 속도와 최소 재고선을 플레이어가 설정한다.
- UI 정보량이 많다: 총비용·현재 병목·효과 요약을 1차 화면에, 상세 흐름을 2차 화면에 둔다.

## 12. 다른 제안과의 연계

- SUG06: 최초 완공과 메타 불가사의가 환생 포인트·유산을 제공한다.
- SUG07: 제한 시간, 특정 문명, 최소 건물 조건의 불가사의 도전을 만든다.
- SUG08: 문명별 건설비·해금·고유 불가사의를 제공한다.
- SUG09: 국가 설립 조건이나 완공 후 국가 고유 modifier를 연결한다.
- SUG10: 정부·교육·복지 체계가 해금과 유지비에 영향을 준다.
- SUG11: 건설 단계, 유적 발견, 완공 축제, 유지 위기 이벤트를 제공한다.
- SUG13: 기술자·건축가·정치가 위인이 비용·진행·사건 결과를 바꾼다.

## 13. 추가 결정 필요

- 불가사의 납품 수요를 시장 가격 계산에 포함할지
- 동시 프로젝트 수와 두 번째 프로젝트 페널티
- 취소 환급률·국고 유지비·진행 정지 규칙
- 최초 완공과 반복 완공의 메타 보상 차이
- 2D 카드 중심 초기 구현과 향후 씬 시각화의 경계
