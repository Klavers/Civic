# SUG09 — 조건부 문명·국가 설립

> 상태: 제안
> 연계: SUG06 환생, SUG07 도전과제, SUG08 시작 문명, SUG10 정치·사회 체계, SUG11 이벤트, SUG12 불가사의, SUG13 위인

## 1. 제안 목적

런 도중 특정 조건을 만족하면 시작 문명을 새로운 문명 또는 국가로 발전시키는 설립 기능을 도입한다. 시작 문명은 출신 정체성으로 남고, 설립 국가는 현재 런의 정치·경제 정체성과 modifier를 결정한다.

국가 설립은 단순 이름 변경이 아니라 다음을 함께 제공한다.

- 국가 고유 modifier와 제약
- 정치·사회 체계의 초기 상태 또는 특별 선택지
- 전용 도전과제·이벤트·불가사의·위인 해금
- 환생 후 시작 문명 또는 유산 해금 조건

## 2. 설립 절차 권장안

1. `발견`: 필수 조건의 70% 이상을 충족하면 후보 카드가 나타난다.
2. `선언`: 기술·시대·출신 문명·경제 조건을 모두 만족하고 국고를 지불한다.
3. `준비`: 60~180초 동안 GDP, 인구, 정책 또는 공급률 조건을 유지한다.
4. `선택`: 국가 고유 헌장 2~3개 중 하나를 선택한다.
5. `통합`: modifier, 이벤트, 도전과제, 명칭과 UI 테마를 적용한다.

준비 중 조건이 깨져도 즉시 실패시키지 않고 진행을 일시 정지한다. 장기간 미충족 시에만 준비가 서서히 감소하도록 해 순간 품귀로 긴 준비가 무효화되는 좌절을 줄인다.

장점은 시작 문명 선택이 런 중반 목표로 이어진다는 점이다. 단점은 단일 정답 경로가 될 수 있어 각 시작 문명에 최소 2개 후계 후보가 필요하다.

장점은 기술 선택이 국가 정체성으로 발전한다는 점이다. 단점은 연구 완료만으로 쉽게 달성되지 않도록 실제 생산·공급 조건을 함께 요구해야 한다.

|-|-|-|-|
| 1. 대도시 동맹 | 고전시대 이상, 인구 500, 집 100개, 건물/인구 비율 90% 이상 | 집 인구 보너스 +1, 건설비 +5%; 도시 이벤트와 `백만의 광장` 해금 | 높은 인구를 활용하지만 포화된 건물 슬롯과 자원 소비를 관리한다. |

장점은 인구가 단순 건물 한도를 넘어 국가 형성 목표가 된다는 점이다. 단점은 생활수준 시스템이 없으면 일부 경로가 비활성화되므로 feature dependency를 명시해야 한다.

## 8. 국가 modifier와 정체성 규칙

- 시작 문명 특성과 설립 국가 modifier는 함께 유지한다.
- 같은 modifier group은 가산 후 상한을 적용하고, 국가 modifier가 시작 문명을 완전히 대체하지 않는다.
- 한 런의 `현재 국가`는 하나만 활성화한다.
- 상위 국가로 재설립할 수 있으나 이전 국가의 강한 modifier는 제거하고 `역사 유산` 1개만 선택해 유지한다.
- 국가 설립 시 건물·자원·기술을 삭제하지 않는다. 정치·경제 규칙과 해금만 변경한다.
- 설립 취소는 준비 단계에서 가능하지만 완료 후에는 다음 상위 국가 설립 또는 환생 전까지 되돌리지 못한다.

## 9. UI 제안

- 국가 패널은 `현재 정체성`, `발견한 후보`, `숨은 후보 힌트`, `설립 준비` 탭으로 구성한다.
- 후보 카드에 필수 조건, 현재값, 유지시간, 설립 비용, modifier, 해금 콘텐츠를 표시한다.
- 조건이 거의 충족되면 HUD 알림에서 가장 부족한 1~3개 조건을 안내한다.
- 설립 준비 중에는 진행 막대와 정지 원인을 표시하고 관련 자원·건물·정책 패널로 이동할 수 있게 한다.
- 완료 화면에서 국기·색상·연대기 문장과 함께 헌장 선택지를 보여준다.
- 국가명·상징은 데이터 에셋으로 관리해 사용자가 Editor에서 교체할 수 있게 한다.

## 10. 플레이 경험 장단점

### 장점

- 런 중반과 후반에 명확한 복합 목표를 제공한다.
- 시작 문명, 기술, 경제, 정치, 이벤트를 하나의 국가 서사로 결합한다.
- 같은 시작 문명에서도 다른 국가로 발전할 수 있어 재플레이가 증가한다.

### 단점 및 완화

- 조건이 많으면 체크리스트 게임이 될 수 있다: 필수 3~5개와 선택 보너스 조건을 분리한다.
- 설립 modifier가 지나치게 강하면 반드시 거쳐야 한다: 장점과 제약을 함께 둔다.
- 숨은 국가는 공략 의존도가 높다: 런 중 일부 조건 충족 시 단계적 힌트를 제공한다.
- 데이터 간 순환 의존이 생길 수 있다: 국가 → 도전 → 문명 해금 방향을 명시하고 validator로 순환을 검사한다.

## 11. 데이터 모델 제안

| 데이터 | 주요 필드 | 역할 |
|-|-|-|

## 12. 다른 제안과의 연계

- SUG06: 국가 설립 기록을 환생 포인트와 유산 해금에 반영한다.
- SUG07: 특수 국가 설립·제약 설립·다단계 설립을 도전과제로 만든다.
- SUG08: 시작 문명별 후계 국가와 범용 기술 국가를 함께 제공한다.
- SUG10: 정책 조합을 설립 조건과 국가 초기 헌장으로 사용한다.
- SUG11: 설립 선언·반대·통합 이벤트와 비밀 국가 이벤트를 제공한다.
- SUG12: 특정 불가사의를 설립 조건 또는 국가 고유 건축으로 연결한다.
- SUG13: 건국 위인이 준비 속도·헌장 선택·통합 이벤트에 영향을 준다.

## 13. 추가 결정 필요

- 한 런의 국가 설립 가능 횟수와 국가 tier 수
- 준비 단계 기본 시간과 조건 이탈 시 감소 규칙
- 시작 문명 특성과 국가 modifier의 합산 상한
- 실제 역사 국가명 사용 여부와 fictional 세계관 범위
- 국가 설립이 저장·불러오기와 환생 기록에 미치는 계약
