# SUG11 — 조건부 이벤트와 사건 연쇄

> 상태: 제안
> 연계: SUG06 환생, SUG07 도전과제, SUG08 시작 문명, SUG09 국가 설립, SUG10 정치·사회 체계, SUG12 불가사의, SUG13 위인

## 1. 제안 목적

기술 연구, 시대 진입, 자원 품귀·과잉, 국가 설립, 정치 개혁, 환생 포인트, 도전과제, 불가사의, 위인 상태를 조건으로 게임 내 사건을 발생시킨다. 이벤트는 숫자 보너스를 무작위로 지급하는 팝업이 아니라 플레이어가 현재 경제와 장기 목표를 바탕으로 선택하는 단기 의사결정이다.

이벤트는 세 종류로 나눈다.

- `확정 이벤트`: 최초 기술 연구·시대 진입·국가 설립처럼 조건 충족 즉시 1회 발생
- `조건부 확률 이벤트`: 품귀·과잉·낮은 정당성 같은 상태를 유지할 때 가중치가 누적
- `연쇄 이벤트`: 첫 선택이 후속 조건과 결말을 만든 2~5단계 사건

## 2. 발생과 운 보정 규칙

- 이벤트 판정은 매 frame이 아니라 5~10초 간격의 event scheduler에서 수행한다.
- 각 이벤트는 필수 조건, 금지 조건, 기본 가중치, 상태 가중치, cooldown, run당 최대 횟수를 가진다.
- 조건부 이벤트가 오랫동안 발생하지 않으면 `누적 가중치`가 증가하고 발생 후 초기화된다.
- 도전과제에 필요한 이벤트는 조건을 충분히 오래 유지하면 확정 발생하는 pity threshold를 둔다.
- 저장·불러오기 반복으로 결과를 바꾸지 못하도록 런 seed와 event sequence로 난수를 결정한다.
- 이벤트가 뜬 동안 시뮬레이션을 멈출지 여부는 설정으로 제공하되 기본값은 일시정지다.

## 3. 대분류 1 — 기술과 시대 발견 이벤트

새 기술·시대가 생산 체인과 사회에 미치는 변화를 선택으로 보여준다.

| 이벤트 | 발생 조건 | 선택·효과 | 플레이 시나리오 |
|-|-|-|-|
| 1. 첫 인쇄물 | 인쇄술 최초 연구, 종이 공방 보유 | `관보`: 정치력 +20·종이 수요 증가 / `교본`: 과학 +10% 120초 / `상업 전단`: GDP +5% 120초 | 종이 공급이 충분한지 보고 단기 과학·정치·경제 중 하나를 선택한다. |
| 2. 증기의 약속 | 증기기관 연구, 석탄 공급률 80% 이상 | `공장 우선`: 기계부품 +20%·생활수준 -5 / `건설 우선`: 건설력 +20% / `안전 규정`: 생산 +10%·국고 지출 | 산업 가속과 사회 비용 사이에서 런 방향을 정한다. |
| 3. 정보의 홍수 | 정보시대 진입, 학교 15개 이상 | `개방망`: 과학 +15%·정당성 변동 / `통제망`: 이벤트 예측 +1·정치력 +10 / `상업망`: 고가 상품 가격 하한 증가 | 후기 국가의 정치체계와 목표에 맞는 정보 운영 방식을 선택한다. |

장점은 기술 연구 결과를 즉시 체감하게 한다. 단점은 매 기술마다 팝업이 뜨면 피로하므로 시대당 핵심 기술 2~3개만 확정 이벤트를 갖는다.

## 4. 대분류 2 — 자원 품귀와 과잉 이벤트

수요·공급·재고 상태를 단기 위기와 기회로 바꾼다.

| 이벤트 | 발생 조건 | 선택·효과 | 플레이 시나리오 |
|-|-|-|-|
| 1. 빵줄의 소문 | 식량 공급률 50% 미만 60초, 인구 100 이상 | `배급`: 소비 -20%·GDP -5% / `긴급 수입`: 국고 지출·밀 재고 획득 / `시장 방치`: 정당성 하락·가격 이익 | 국고와 정치 안정 중 무엇을 지킬지 선택한다. |
| 2. 목재 범람 | 나무 재고가 수요의 20배, 가격 최저 구간 120초 | `공공 건축`: 건설력 획득 / `가구 장려`: 가구 공방 조건부 효과 / `비축 유지`: 이후 불가사의 비용 할인 | 과잉 재고를 성장 자원으로 전환한다. |
| 3. 석탄의 검은 겨울 | 석탄 수요가 공급의 2배, 공장 20개 이상 | `광산 보조금`: 국고 지출·석탄 생산 +20% / `공장 휴업`: 소비 감소·GDP 하락 / `대체 연료 연구`: 과학 소비·특수 기술 효과 | 품귀를 단순 경고가 아닌 산업 정책 선택으로 처리한다. |

장점은 자원 균형에 서사를 부여한다. 단점은 위기 이벤트가 이미 불리한 상태를 더 악화시킬 수 있어 최소 하나의 회복 선택지를 반드시 제공해야 한다.

## 5. 대분류 3 — 국가 설립과 정치 개혁 이벤트

국가·정책 변화 과정에서 이해집단의 반응과 헌장 선택을 발생시킨다.

| 이벤트 | 발생 조건 | 선택·효과 | 플레이 시나리오 |
|-|-|-|-|
| 1. 건국회의 | SUG09 국가 설립 준비 50% 도달 | `중앙집권`: 준비 속도 +20%·저항 +15 / `연방타협`: 속도 -10%·완료 정당성 +15 / `상인 후원`: 국고 지출·GDP modifier | 빠른 건국과 장기 안정 사이에서 헌장 방향을 정한다. |
| 2. 투표권 청원 | 재산권 선거, 생활수준 70, 보통선거 해금 | `수용`: 개혁 진행 +25 / `위원회`: 정치력 획득·진행 정지 / `거부`: 정당성 -15·상공업 지지 +5 | 현재 국가의 지지 기반을 보고 개혁 시기를 결정한다. |
| 3. 복지 예산 파동 | 보편 사회보장 제안 중 국고 증가량이 지출 예상보다 낮음 | `세율 인상`: 세수 +0.02·GDP -3% / `축소안`: 복지 효과 -20% / `국채 이벤트`: 즉시 국고·후속 위기 | 지속 가능한 제도와 단기 개혁 성공 중 선택한다. |

장점은 개혁 진행도가 단순 타이머가 되는 것을 막는다. 단점은 선택 결과가 불투명하면 불공정하게 느껴지므로 예상 효과 범위를 명시해야 한다.

## 6. 대분류 4 — 환생·도전과제·메타 이벤트

여러 런의 기록과 환생 포인트에 따라 새 사건을 해금한다.

| 이벤트 | 발생 조건 | 선택·효과 | 플레이 시나리오 |
|-|-|-|-|
| 1. 이전 문명의 꿈 | 환생 3회 이상, 새 런 원시시대 | `경제의 기억`: 시작 자원 / `지식의 기억`: 과학 / `통치의 기억`: 정치력 중 하나를 선택 | 같은 유산 구성에서도 매 런 다른 초기 보조를 선택한다. |
| 2. 도전자의 초대 | 고난도 도전과제 5개 완료, 현재 런 도전 미추적 | 특수 제약 도전을 수락하면 환생 포인트 배율 증가, 거절하면 cooldown | 플레이어가 자발적으로 위험한 런 목표를 추가한다. |
| 3. 연대기의 빈 페이지 | 이벤트 도감 70%, 비밀 이벤트 미발견 | 힌트를 선택해 특정 조건 가중치를 올리거나 환생 포인트를 지불해 확정 단서를 얻음 | 수집 막바지의 무작위 탐색 피로를 줄인다. |

장점은 메타 진척이 새 이벤트와 선택으로 체감된다는 점이다. 단점은 환생 포인트를 지불하는 선택이 강제처럼 느껴지지 않도록 무료 우회 경로를 둔다.

## 7. 대분류 5 — 불가사의와 위인 연쇄 이벤트

거대 건축과 인물의 생애를 사건 연쇄로 연결한다.

| 이벤트 | 발생 조건 | 선택·효과 | 플레이 시나리오 |
|-|-|-|-|
| 1. 기초석 아래의 유적 | 불가사의 건설 25%, 고대시대 이상 | `발굴`: 건설 지연·과학 획득 / `보존`: 비용 증가·완공 명성 보너스 / `덮기`: 즉시 진행 | 단기 완공과 장기 보상 사이에서 선택한다. |
| 2. 위인의 경쟁 | 같은 역할 위인 2명 활성, 슬롯 한도 도달 | 한 명 후원, 공동 프로젝트, 경쟁 방치 중 선택해 충성도·성과·이탈 가능성 변화 | 강한 인물 둘을 유지할지 한 명에게 자원을 집중할지 결정한다. |
| 3. 마지막 설계도 | 기술자 위인 은퇴 임박, 불가사의 건설 중 | `완공 헌신`: 위인 조기 은퇴·진행도 +20 / `후계 양성`: 후속 위인 확률 / `회고록`: 영구 도감 보상 | 위인의 생애 마무리를 현재 건축 목표와 연결한다. |

장점은 인물과 불가사의가 정적인 modifier가 아니라 이야기 주체가 된다는 점이다. 단점은 이벤트가 너무 자주 개입하지 않도록 위인당 핵심 연쇄 1개로 제한한다.

## 8. 이벤트 선택 결과 규칙

- 결과는 `즉시 효과`, `기간 modifier`, `후속 플래그`, `관계·정당성 변화`, `도감 기록`으로 구성한다.
- 모든 선택에는 최소 하나의 명시적 비용 또는 기회비용을 둔다.
- 동일 이벤트의 반복 발생은 선택 결과가 누적 폭주하지 않도록 stack cap을 가진다.
- 기간 modifier는 sourceId와 만료 tick을 저장하고 저장·불러오기 후에도 정확히 복원한다.
- 후속 이벤트는 이전 선택 ID를 조건으로 참조하고 순환 체인은 validator가 차단한다.

## 9. UI 제안

- 이벤트 창은 제목, 삽화 슬롯, 발생 원인, 현재 관련 수치, 선택지, 예상 효과를 표시한다.
- 발생 원인을 `석탄 공급률 43%가 60초 지속됨`처럼 구체적으로 보여준다.
- 각 선택지 tooltip에 즉시 효과, 기간, 후속 가능성, 도전과제 영향 여부를 분리 표시한다.
- 이벤트 도감은 발견/미발견/연쇄 진행 상태와 마지막 선택을 기록한다.
- HUD의 사건 대기열은 최대 3개로 제한하고, 긴급 이벤트만 자동 일시정지한다.
- 접근성 옵션으로 확률·정확한 수치·숨은 조건 표시 수준을 선택할 수 있게 한다.

## 10. 플레이 경험 장단점

### 장점

- 동일한 생산 상태에서도 선택에 따라 런의 이야기가 달라진다.
- 품귀·기술·개혁 같은 시스템 변화가 숫자 외의 체감으로 전달된다.
- 도감과 연쇄 사건이 여러 환생의 수집 목표가 된다.

### 단점 및 완화

- 팝업 피로: 발생 빈도 상한, 자동 선택 preset, 중요도별 일시정지를 제공한다.
- 운에 따른 도전 실패: 누적 가중치와 확정 발생 threshold를 둔다.
- 결과 예측 불가: 예상 범위와 후속 가능성을 선택 전에 표시한다.
- 저장 반복 악용: deterministic run seed와 event sequence를 사용한다.

## 11. 데이터 모델 제안

| 데이터 | 주요 필드 | 역할 |
|-|-|-|
| `events.csv` | id, category, titleKo, descriptionKo, triggerMode, baseWeight, cooldown, maxPerRun | 사건 정의 |
| `event_conditions.csv` | eventId, conditionType, targetId, comparator, value, duration, requiredFlags | 발생 조건 |
| `event_choices.csv` | eventId, choiceId, textKo, requirementId, nextEventId | 선택지 |
| `event_effects.csv` | choiceId, effectType, targetId, amount, duration, stackGroup | 결과 |
| `EventSchedulerState` | seed, sequence, cooldowns, accumulatedWeights, queuedIds | 런 상태 |
| `EventHistory` | eventId, choiceId, occurredAt, runId | 도감·연대기 |

조건 평가는 SUG07·SUG09·SUG10과 같은 condition evaluator를 공유한다. 선택 효과는 modifier pipeline과 resource flow ledger에 sourceType=`event`로 전달한다.

## 12. 다른 제안과의 연계

- SUG06: 환생 횟수·유산·포인트에 따른 메타 이벤트를 해금한다.
- SUG07: 이벤트 발견·특정 선택·연쇄 결말을 도전과제로 사용한다.
- SUG08: 시작 문명별 전용 사건과 약점 보완 사건을 제공한다.
- SUG09: 국가 설립 준비와 비밀 국가 발견을 사건 연쇄로 구성한다.
- SUG10: 개혁 진행 구간과 정당성 위기를 사건 조건으로 사용한다.
- SUG12: 건설 단계·완공·파손·발견 사건을 제공한다.
- SUG13: 위인 모집·경쟁·은퇴·유산 사건을 제공한다.

## 13. 추가 결정 필요

- 기본 이벤트 판정 주기와 시간당 목표 발생 수
- 일시정지·대기열·자동선택 기본 정책
- 이벤트 선택 결과를 얼마나 정확히 공개할지
- 확률 이벤트 pity threshold와 도전과제 예외 규칙
- 텍스트·삽화 localization 및 사용자 편집 prefab 구조
