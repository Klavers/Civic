# SUG13 — 위인·인물과 라이프사이클

> 상태: 제안
> 연계: SUG06 환생, SUG07 도전과제, SUG08 시작 문명, SUG09 국가 설립, SUG10 정치·사회 체계, SUG11 이벤트, SUG12 불가사의

## 1. 제안 목적

문명과 Victoria 3처럼 경제·기술·정치·건축에 영향을 주는 위인과 인물을 도입한다. 인물은 영구 modifier 카드가 아니라 생성, 등용, 배치, 활동, 사건, 은퇴·사망, 유산으로 이어지는 라이프사이클을 가진다.

인물 시스템의 목적은 다음과 같다.

- 숫자 성장에 이름과 서사를 부여한다.
- 한정된 슬롯에 누구를 배치할지 선택하게 한다.
- 위인 능력과 현재 경제 병목을 연결한다.
- 은퇴 후에는 일부 유산만 남겨 새 인물을 계속 사용할 이유를 만든다.

## 2. 공통 라이프사이클

### 2.1 생성

- 학자 진척: 과학 생산·기술 연구에서 축적
- 기술자 진척: 건설력 사용·공장·불가사의에서 축적
- 정치가 진척: 국가 설립·개혁·정당성 위기에서 축적
- 기업가 진척: GDP·국고·가격 변동·가공품에서 축적
- 문화가·탐험가 진척: 이벤트 발견·도전과제·새 시대·불가사의에서 축적

진척이 threshold에 도달하면 후보 3명이 생성된다. 후보는 런 seed와 현재 조건에서 결정하되, 같은 후보가 반복 탈락하면 가중치를 낮춘다.

### 2.2 등용과 제한

- 기본 활성 인물 슬롯: 3명
- 역할별 동일 인물 제한: 없음. 다만 같은 효과군은 감쇠
- 후보 대기열: 3명, 120초 유지
- 산업시대 국가·불가사의·유산으로 활성 슬롯 최대 5명
- 등용 비용: 국고 또는 정치력. 희귀 인물은 특정 도전·문명·국가 조건 필요

### 2.3 관리

각 인물은 한 가지 `직책` 또는 `프로젝트`에 배치한다.

- 생산부문 감독: 특정 건물군
- 연구 책임자: 특정 시대 또는 기술군
- 개혁 후원자: 진행 중인 정책
- 건국 사절: 국가 설립 준비
- 불가사의 책임자: 건설 프로젝트

배치 변경에는 짧은 cooldown과 정치력 비용을 둔다. 인물은 상시 특성 1개, 직책 효과 1개, 런당 1회 능동 능력 1개를 가진다.

### 2.4 소멸과 유산

- `임기`: 게임 시간 600~1,800초. 역할·시대·사건에 따라 다름
- 임기 종료 전 은퇴 이벤트가 발생한다.
- 사고·정치 위기로 조기 이탈할 수 있으나 무작위 즉사만으로 제거하지 않는다.
- 은퇴 시 업적 점수에 따라 `유산 카드`를 남긴다.
- 활성 효과는 사라지고 유산 효과의 10~25%만 런 종료까지 유지한다.
- 환생 시 인물과 런 유산은 사라지지만 도감·업적·특정 메타 유산은 유지한다.

## 8. 인물 관리와 슬롯 설계

- 기본 활성 슬롯 3개는 `국가 직책`, `전문 직책`, `프로젝트 직책`으로 나누되 어떤 역할도 배치 가능하게 한다.
- 같은 역할 2명 이상 배치 시 두 번째부터 상시 효과 70%, 세 번째 50%로 감쇠한다.
- 능동 능력은 인물별 런당 1회가 기본이며 일부는 긴 cooldown으로 재사용한다.
- 후보를 거절하면 소량 정치력 또는 해당 역할 진척을 반환해 완전한 손실을 줄인다.
- 인물 해임은 가능하지만 정당성·정치력 비용과 해당 인물의 이탈 이벤트를 발생시킨다.
- 자동 배치는 권장 직책만 제안하고 실제 변경은 플레이어가 확정한다.

## 9. 위인 희귀도와 생성 제어

- 일반 인물: 조건이 넓고 작은 효과, 반복 등장 가능
- 유명 인물: 특정 기술·국가·문명 조건, 런당 1회
- 전설 인물: 도전과제·비밀 이벤트·환생 포인트 단계로 해금, 계정당 발견 기록

희귀도는 단순 수치 배율이 아니라 고유 능동 능력과 사건 연쇄의 차이로 표현한다. 환생 포인트를 지불해 후보를 직접 구매하는 방식은 pay-to-win형 메타가 되므로 비권장하고, 후보 재추첨·힌트·등장 가중치 정도만 허용한다.

## 10. UI 제안

- 인물 패널은 `활성`, `후보`, `프로젝트`, `은퇴자`, `도감` 탭으로 구성한다.
- 인물 카드는 초상화 prefab, 이름, 역할, 임기, 특성, 현재 직책, 능동 능력, 관련 이벤트를 표시한다.
- drag-and-drop 또는 `배치` 버튼으로 직책 슬롯에 할당하고 예상 효과 변화를 미리 보여준다.
- 임기 20% 이하, 능동 능력 사용 가능, 이탈 위험 시 HUD 알림을 표시한다.
- 은퇴 화면은 해당 런의 기여 통계와 선택 가능한 유산 2개 중 하나를 보여준다.
- 초상화와 아이콘은 Addressable 또는 데이터 참조로 교체 가능하게 하고, UI 계층은 prefab으로 생성한다.

## 11. 플레이 경험 장단점

### 장점

- 경제 변화에 인물 이름과 기억할 사건을 부여한다.
- 제한 슬롯으로 현재 병목에 맞는 인물을 선택하는 전략이 생긴다.
- 은퇴와 유산으로 강력한 인물이 영구 고착되지 않고 세대가 교체된다.
- 다른 제안 시스템을 연결하는 공통 서사 장치가 된다.

### 단점 및 완화

- 관리 피로: 기본 3명, 최대 5명으로 제한하고 자동 추천을 제공한다.
- 무작위 후보 불만: 누적 가중치, 재추첨, 조건 힌트, 동일 후보 반복 방지를 둔다.
- 인물 상실 좌절: 조기 이탈 전 경고와 대응 이벤트를 제공하고 일부 유산을 보장한다.
- modifier 출처 복잡성: 자원·건물 상세에서 인물 기여를 별도 source로 표시한다.

## 12. 데이터 모델 제안

| 데이터 | 주요 필드 | 역할 |
|-|-|-|
| `people.csv` | id, displayNameKo, archetypeId, rarity, portraitId, baseTenure | 인물 정의 |
| `person_conditions.csv` | personId, conditionType, targetId, comparator, value, weight | 후보 조건 |
| `person_traits.csv` | personId, traitType, targetId, amount, capGroup | 상시 특성 |
| `person_assignments.csv` | assignmentId, allowedArchetypes, effectType, targetScope | 직책 정의 |
| `person_abilities.csv` | personId, abilityId, costType, effectType, amount, duration, usesPerRun | 능동 능력 |
| `person_events.csv` | personId, lifecycleStage, eventId, weight | 사건 연계 |
| `PersonRuntimeState` | personId, tenureRemaining, assignmentId, abilityUses, statusFlags, contributionStats | 런 상태 |
| `PersonLegacyRecord` | personId, discovered, endings, bestContribution, unlockedLegacyIds | 메타 도감 |

인물 효과는 sourceType=`person`, sourceId를 가진 modifier로 계산한다. 임기와 능력 cooldown은 simulation tick 기준으로 저장해 오프라인 진행 도입 여부와 독립적으로 재현 가능해야 한다.

## 13. 다른 제안과의 연계

- SUG06: 위인 연대기 유산, 후보 갱신, 발견 기록을 메타 보상으로 연결한다.
- SUG07: 특정 인물 조합·은퇴 결말·무위인 런을 도전과제로 만든다.
- SUG08: 시작 문명별 인물 역할 가중치와 전용 인물을 제공한다.
- SUG09: 건국 사절·헌장 작성자·비밀 국가 인물을 설립 과정에 연결한다.
- SUG10: 정치가·개혁가·학자가 지지, 정당성, 개혁 속도를 바꾼다.
- SUG11: 모집·경쟁·위기·은퇴·유산을 사건 연쇄로 제공한다.
