# SUG06 — 환생과 문명 유산

> 상태: 제안
> 연계: SUG07 도전과제, SUG08 시작 문명, SUG09 국가 설립, SUG10 정치·사회 체계, SUG12 불가사의, SUG13 위인

## 1. 제안 목적

한 번의 문명 플레이를 끝내고 새 문명을 시작하는 환생 기능을 도입한다. 환생 시 해당 런에서 기록한 최고 GDP, 최고 인구, 도달 시대, 연구한 기술, 도전과제 성과를 환생 포인트로 환산한다. 플레이어는 환생 포인트로 영구 보너스인 `문명 유산`을 구매하며, 다음 런에서는 다른 시작 문명과 다른 성장 경로를 시도한다.

핵심 목적은 단순 수치 초기화가 아니라 다음 세 가지다.

- 한 런의 종료 시점을 플레이어가 스스로 판단하게 한다.
- 장기적으로 새로운 시작 문명·도전·정책 조합을 시험할 이유를 만든다.
- 완전 초기화의 상실감을 영구 진척과 기록 보존으로 상쇄한다.

## 2. 환생 포인트 산정 권장안

환생 포인트는 순간값이 아니라 런 최고 기록을 기준으로 계산한다. 저장 직전 자원을 소모하거나 건물을 철거해 점수가 흔들리는 문제를 막기 위해서다.

```text
환생 포인트 = GDP 점수 + 인구 점수 + 시대 점수 + 기술 점수 + 도전 점수

GDP 점수   = floor(log10(런 최고 GDP + 1) × 4)
인구 점수  = floor(sqrt(런 최고 인구))
시대 점수  = 도달한 최고 시대 order × 5
기술 점수  = Σ(연구 기술의 시대 가중치) / 5
도전 점수  = 해당 런에서 최초 달성한 도전과제 보상
```

위 수치는 구현 확정값이 아니라 첫 밸런스 테스트용 기준이다. GDP와 인구에는 로그·제곱근을 사용해 같은 구간 반복 파밍의 효율이 점차 낮아지게 한다. 시대와 기술은 새로운 콘텐츠에 도달할수록 점수가 크게 증가하도록 구성한다.

### 환생 시 초기화·유지 범위

| 구분 | 초기화 | 유지 | 이유 |
|-|-|-|-|
| 경제 | 자원 재고, 건물 수, GDP, 국고, 건설력 | 최고 기록 | 새 런의 경제 퍼즐 복원 |
| 기술 | 연구 완료 상태, 현재 시대 | 기술 도감, 최고 도달 시대 | 연구 선택을 다시 경험 |
| 사회 | 정책, 국가, 활성 이벤트 상태 | 해금된 선택지와 연대기 | SUG09~SUG11 연계 |
| 메타 | 없음 | 환생 포인트, 유산, 도전과제, 시작 문명 해금 | 장기 진척 정본 |
| 인물·불가사의 | 활성 위인과 건설물 | 발견 기록, 영구 보상만 | 런별 희소성 유지 |

환생은 최소 고대시대 진입 후 가능하게 하고, 예상 획득 포인트가 0이면 확인 경고를 표시한다. 자동 환생은 초기 범위에서 제외한다.

## 3. 대분류 1 — 경제 유산

경제 유산은 생산·건설·재고 운영을 안정화한다. 직접적인 무한 배율보다 초반 병목을 줄이는 방향을 권장한다.

| 예시 | 영구 효과 | 예시 시나리오 |
|-|-|-|
| 1. 개척 건설단 | 새 런 시작 건설력 +5, 3단계까지 증가 | 첫 환생 후 벌목장과 밀농장을 즉시 확보해 반복 초반을 단축한다. |
| 2. 표준화된 공구 | 원시·고대 건물 건설비 -3%/단계 | 산업 문명으로 시작하되 초반 공구 병목을 줄여 전문 생산 체인에 빨리 진입한다. |
| 3. 비축 관행 | 시작 시 해금된 기본 자원 재고 +2/단계 | 인구 소비가 시작되기 전 밀·직물 완충 재고를 확보해 첫 품귀를 완화한다. |

플레이 경험 장점은 반복 구간 단축과 성장 체감이다. 단점은 누적 보너스가 경제 퍼즐을 무력화할 수 있다는 점이다. 각 효과에 단계 상한을 두고, 후반 생산 배율보다 초반 편의에 집중해야 한다.

## 4. 대분류 2 — 인구와 정착 유산

인구 유산은 시작 인구, 인구 소비 안정성, 주거 운영을 보조한다.

| 예시 | 영구 효과 | 예시 시나리오 |
|-|-|-|
| 1. 개척민 전통 | 시작 인구 +1/단계, 최대 3 | 건물 슬롯이 부족한 시작 문명도 초반 생산건물을 하나 더 배치할 수 있다. |
| 2. 공동 저장고 | 첫 60초 동안 인구층 기본 소비 요구량 -10% | 새 문명의 생산 체인을 정비할 시간을 확보하되 장기 소비량은 정상으로 돌아온다. |
| 3. 도시 설계 기억 | 집 계열 건물의 조건 충족 보너스 판정에 5% 완충률 | 짧은 자원 변동으로 인구 보너스가 반복 취소되는 체감을 줄인다. |

장점은 환생 직후 정착 실패를 줄이는 것이다. 단점은 인구 한도를 너무 쉽게 늘리면 건물 선택의 긴장이 약해진다. 시작 인구와 주거 보너스에는 강한 상한이 필요하다.

## 5. 대분류 3 — 지식과 연구 유산

연구 유산은 과학 획득과 기술 탐색을 보조하지만 연구 완료 상태 자체를 그대로 계승하지 않는다.

| 예시 | 영구 효과 | 예시 시나리오 |
|-|-|-|
| 1. 구전 지식 | 시작 과학 +3/단계 | 시작 기술 1개를 더 빨리 선택해 문명별 빌드를 차별화한다. |
| 2. 시대 기록보관소 | 이전 런 최고 시대보다 낮은 시대 기술 비용 -2%/단계 | 반복한 원시·고대 구간은 빨라지지만 새 최고 시대의 연구 비용은 그대로 유지된다. |
| 3. 연구 방법론 | 각 시대 첫 연구 완료 시 과학 일부 환급 | 어떤 기술을 첫 기술로 고를지에 따라 초반 연구 순서가 달라진다. |

장점은 이미 경험한 구간의 반복 피로를 줄이면서 새 시대 도전은 보존한다는 점이다. 단점은 기술 비용 감소가 누적되면 시대 진행 규칙이 무의미해질 수 있어 최고 시대 이하에만 적용하는 조건이 필요하다.

## 6. 대분류 4 — 제도와 통치 유산

제도 유산은 SUG09 국가 설립과 SUG10 정치·사회 체계 변경 비용을 완화한다.

| 예시 | 영구 효과 | 예시 시나리오 |
|-|-|-|
| 1. 행정 문서 양식 | 정책 변경 행정력 또는 국고 비용 -5%/단계 | 선거제와 교육제도를 자주 실험하는 런에서 개혁 비용을 절약한다. |
| 2. 건국 신화 | 국가 설립 조건 중 하나의 유지 시간 -10% | 목표 수치를 순간 달성하는 것이 아니라 짧게 유지해도 건국 준비가 진전된다. |
| 3. 조세 관행 | 시작 세율 +0.01, 최대 +0.03 | 국고 부족으로 건설부문이 멈추는 초반 위험을 완화한다. |

장점은 정치·국가 시스템을 환생 메타와 연결하는 것이다. 단점은 조세 보너스가 GDP 성장과 곱해져 과도해질 수 있으므로 가산값과 상한을 사용해야 한다.

## 7. 대분류 5 — 도전과 문화 유산

도전 유산은 특정 플레이 방식과 수집 목표를 장기 보상으로 연결한다.

| 예시 | 영구 효과 | 예시 시나리오 |
|-|-|-|
| 1. 불가사의 장인단 | 환생당 첫 불가사의 건설비 -10% | SUG12의 거대 건축을 매 런 하나씩 시도할 동기를 준다. |
| 2. 위인 연대기 | 위인 후보 갱신 비용 1회 무료 | SUG13에서 원하는 전문 인물을 확보할 확률을 높인다. |
| 3. 도전자 인장 | 선택한 고난도 도전과제 완료 시 환생 포인트 +20% | 평범한 성장 런 대신 특정 제약을 감수하는 도전 런을 선택한다. |

장점은 환생이 단순 생산 배율 구매가 아니라 다른 시스템을 탐색하는 동기가 된다는 점이다. 단점은 특정 콘텐츠를 강제할 수 있으므로 보너스 경로를 여러 개 제공해야 한다.

## 8. 환생 예시 시나리오

플레이어가 산업시대, 최고 GDP 12,000, 최고 인구 225, 기술 58개, 도전과제 2개를 달성했다. 환생 화면은 항목별 점수, 총 획득 포인트, 다음 유산 구매 가능 여부를 미리 보여준다. 플레이어는 `시대 기록보관소`와 `개척 건설단`을 구매한 뒤 SUG08의 해양 상업 문명을 선택한다. 다음 런에서는 기존보다 원시 구간은 빨라지지만, 해양 문명의 낮은 기초 생산 때문에 새로운 병목이 발생한다.

## 9. UI 제안

- 상단 메뉴에 `문명 유산` 버튼을 두고 현재 포인트와 미사용 포인트를 표시한다.
- 환생 화면은 `현재 런 기록`, `예상 환생 포인트`, `초기화 항목`, `유지 항목`, `다음 시작 문명`의 5영역으로 나눈다.
- 각 점수 항목에 마우스를 올리면 계산 근거와 다음 1포인트까지 필요한 목표를 표시한다.
- 환생 확정은 2단계 확인을 사용하고, 저장 슬롯 또는 자동 백업을 만든 뒤 수행한다.
- 유산 트리는 5개 대분류 탭과 단계형 노드로 표시하며, 구매 취소는 환생 전 미리보기 단계에서만 허용한다.

## 10. 데이터 모델 제안

| 데이터 | 주요 필드 | 저장 범위 |
|-|-|-|
| `prestige_rules.csv` | scoreType, curveType, coefficient, cap | 정적 데이터 |
| `legacy_perks.csv` | id, category, maxRank, costPerRank, effectType, amount | 정적 데이터 |
| `CivicRunRecord` | peakGdp, peakPopulation, highestEraOrder, researchedIds, completedChallengeIds | 런 저장 |
| `CivicMetaProgress` | prestigePoints, perkRanks, unlockedCivilizations, achievementIds | 계정/메타 저장 |
| `PrestigePreview` | componentScores, total, warnings, nextThresholds | 계산 snapshot |

점수 계산은 UI가 아니라 순수 메타 진행 코어에서 수행해야 한다. 저장 데이터에는 계산 결과뿐 아니라 계산 근거가 된 런 기록을 함께 남겨 밸런스 규칙 변경 시 재계산 가능하게 한다.

## 11. 다른 제안과의 연계

- SUG07: 도전과제 최초 달성 보상과 고난도 런 배율을 환생 포인트에 연결한다.
- SUG08: 환생 화면에서 해금된 시작 문명을 선택한다.
- SUG09: 설립한 국가를 메타 도감과 시작 문명 해금 조건으로 사용한다.
- SUG10: 정치·사회 개혁 횟수나 특정 체제 유지 기록을 점수 또는 유산 조건으로 사용한다.
- SUG11: 환생 포인트 단계에 따라 메타 이벤트 체인을 해금한다.
- SUG12: 불가사의 완공 기록을 환생 점수·유산 해금 조건으로 사용한다.
- SUG13: 위인의 업적을 연대기에 남기고 관련 유산을 해금한다.
