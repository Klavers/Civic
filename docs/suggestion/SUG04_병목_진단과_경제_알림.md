# SUG04 — 병목 진단과 경제 알림

## 제안 요약

현재 자원 상세의 생산·소비 흐름을 이용해 “무엇이 부족한가”뿐 아니라 “왜 부족하고 무엇을 바꾸면 되는가”를 알려주는 병목 진단 패널을 추가한다.

## 현재 문제

- 품귀 알림과 공급률은 상태를 보여주지만, 자원이 많아질수록 원인을 찾는 데 여러 패널을 오가야 한다.
- 조건부 기술 효과가 늘어나면 하나의 투입 부족이 여러 건물의 효율을 동시에 낮출 수 있다.
- 숫자만 보고 최적 대응을 계산해야 하면 초반에는 난도가 높고 후반에는 확인 비용이 커진다.

## 권장 구현

시뮬레이션 snapshot에 새로운 계산을 넣기보다 기존 자원 흐름을 읽는 진단 계층을 둔다.

1. 공급률이 가장 낮은 자원 3개를 찾는다.
2. 해당 자원을 가장 많이 소비하는 건물과 영향을 받는 산출을 연결한다.
3. 건물 1개 추가 시 예상 공급 개선량과 필요한 건설력·인구를 표시한다.
4. 알림은 `심각`, `주의`, `기회`로 구분한다.
5. 알림 클릭 시 관련 자원 상세 또는 건물 행으로 이동한다.

예시 문구:

> 석탄 공급률 42% — 의류 공방 8개와 공구공장 5개의 조건부 효과가 감소했습니다. 석탄 광산 +2 건설 시 예상 공급률 78%.

## 플레이 시나리오

방직기 연구 직후 의류 공방의 석탄 수요가 생긴다. 플레이어는 의류 생산이 기대만큼 늘지 않은 이유를 직접 계산할 필요 없이 알림에서 석탄 병목과 영향받는 건물을 확인한다. 석탄 광산 건설 버튼으로 이동해 공급을 보강하면 알림 단계가 즉시 내려간다.

## 기대 효과와 위험

- 복잡한 생산 체인을 학습 가능한 문제로 바꾼다.
- 자원 상세·건물 패널·기술 효과가 하나의 플레이 흐름으로 연결된다.
- 시스템이 정답을 대신 선택하지 않도록 자동 건설은 하지 않고 원인과 예상치만 제공해야 한다.

## 후속 결정 필요

- 심각도 임계값
- 예상치 계산에 재고 버퍼를 포함할지 여부
- 동시에 표시할 알림 수
- 진단 결과의 갱신 주기와 성능 예산
