# session-handoff

세션 연속성을 유지하기 위한 **핸드오프 메모리 파일**을 생성한다. 다음 세션이 이 파일을 읽는 즉시 이전 작업 맥락을 완전히 흡수하도록 설계.

## 왜 이 스킬이 필요한가

세션이 compact 또는 종료되면 맥락이 끊겨 다음 세션이 **같은 실수를 반복**하거나 **사용자 성향을 모른 채 응답**한다. 이 스킬은 그 간극을 메운다. 핸드오프 파일은 git 추적 대상으로, 명시 경로로 staging 하여 다음 PR 사이클에 커밋·정리한다.

## 언제 이 스킬을 사용하나

### 반드시 사용
- "세션 이전 메모리 만들어" / "핸드오프 파일 생성"
- "다음 세션이 이어받을 수 있게 정리"
- "세션 종료 전 메모리 작성"
- 사용자가 명시적으로 "이 세션을 통해서 Codex가 학습한 내용" 요청
- PR 머지 직후 대형 마일스톤 완료 시 (사용자 요청 없어도 핸드오프 권장)

### 사용하지 않음
- AGENTS.md 영구 규범 갱신 (그건 정식 PR 대상)
- plan/ 문서 작성 (그건 `plan-doc` 스킬)

## 저장 위치

```
<repo-root>/.Codex/handoffs/handoff_YYYYMMDD_<topic>.md
```

- 저장 루트는 `scripts/detect_worktree_root.sh` 로 확인 — `git rev-parse --show-toplevel` 결과. worktree 에서 실행하면 worktree 경로, 메인 레포면 메인 레포 경로를 반환하므로 항상 올바른 위치에 생성된다.
- `YYYYMMDD`: 세션 종료일 기준 (today)
- `<topic>`: 세션의 주요 산출물·마일스톤을 영문 snake_case로 요약
  - 예: `handoff_20260516_skill_migration.md`
  - 예: `handoff_20260516_auth_refactor.md`

> `.Codex/handoffs/` 는 git 추적 대상이다. `.Codex/settings.local.json` 만 개인 설정으로 제외되며 handoffs/ 는 정상 tracked.

## 표준 섹션 (7개 고정)

핸드오프 파일은 다음 7개 섹션을 순서대로 포함:

### § 1. 세션 요약
- 세션 시작일 ~ 종료일
- 세션 최대 성과 1~3개
- 머지된 PR 목록 (번호·제목·머지일·내용 한 줄 표)

### § 2. 현재까지 구현된 사항
- 코드/문서 분리: 병합 완료된 작업 범위 + 이번 세션에 반영된 새 문서 목록
- AGENTS.md 갱신된 섹션 이름만 나열 (내용 복사 금지)

### § 3. 향후 수행 우선순위
- **1순위**: 다음 세션이 **바로 시작 가능한** 작업. 상세 spec 포함 (상태·핵심 사실·PR 구조·테스트 전략).
- **2순위 이하**: 표 형식. 순위 \| 작업 \| 의존
- **보류** 항목 명시

### § 4. 사용자 성향 / 세션 운영 스타일
- **이번 세션에서 확립·강화된 것만** 기재 (AGENTS.md 일반 규범은 링크로)
- 꾸준히 요구한 사항, 세션 운영 스타일 특이점

### § 5. 이번 세션에서 학습·기억한 사항
- 프로젝트 구조 핵심 (이번 세션에서 처음 확정된 것)
- 프로젝트 핵심 수치 (단일 소스 포인터)
- 기술 스택 유의사항
- 세션 도구 팁 (gh CLI 경로, 셸 제약 등)

### § 6. 다음 세션 착수 가이드
실제 사용자가 "핸드오프 메모 확인" 지시 시 이 섹션을 따라 행동하도록 **명령형 단계**로 작성:

```markdown
1. AGENTS.md 읽기 (특히 X, Y, Z 섹션)
2. 본 파일 읽기 (.Codex/handoffs/handoff_YYYYMMDD_<topic>.md)
3. 관련 plan 문서 재독
4. 코드 진입점 재확인: `<module>/<file>:NN-MM`
5. 새 feature branch 생성 커맨드
6. 첫 커밋 착수
```

주의사항도 함께 (실수 방지용).

### § 7. 참조 경로 치트시트
자주 참조할 절대 경로 목록 (plan root, 프로젝트 규범 AGENTS.md, 핵심 모듈, 테스트 진입점, 이전 세션 노트).

## 작성 원칙

### 원칙 1: AGENTS.md 중복 회피
핸드오프 메모는 **진행 중 상태 스냅샷**, AGENTS.md는 **영구 규범**. 둘이 중복되면 핸드오프에는 "AGENTS.md 'X 섹션' 참조"만 남기고 규범 문구를 복사하지 않는다.

### 원칙 2: 이번 세션 고유 정보에 집중
핸드오프 메모의 가치는 **이번 세션에서 처음 확정된 것** 또는 **이번 세션에서 강화된 패턴**. 영구 사실은 AGENTS.md가 담당.

### 원칙 3: 다음 세션 착수 가이드가 가장 중요
§6은 핸드오프 파일의 핵심. 여기에 적힌 순서를 따라가면 다음 세션이 **바로 실제 작업에 착수** 가능해야 함. 모호한 지시 금지.

### 원칙 4: 버전관리 포함
`.Codex/handoffs/` 는 git 추적 대상이다. 핸드오프 파일은 명시 경로 `git add` 로 staging 하여 다음 PR 사이클에 포함한다. 효력이 끝난 메모는 다음 PR 에서 정리(또는 영구 가치 시 `docs/` 로 승격).

## 작성 워크플로우

### (1) 세션 정보 수집
- **저장 루트 확인**: `bash .agents/skills/session-handoff/scripts/detect_worktree_root.sh`
- 오늘 날짜: 세션 context의 today
- 머지된 PR 목록: `gh pr list --state merged --limit 5 --json number,title,mergedAt`
- 현재 branch: `git branch --show-current`

### (2) 토픽 결정
세션의 가장 큰 산출물·마일스톤을 영문 snake_case로 요약.

### (3) 파일 생성
`assets/templates/session_handoff.md` 템플릿을 복사하고 채워 넣음. `.Codex/handoffs/` 디렉토리가 없으면 생성.

### (4) 기존 핸드오프 파일 확인 (선택)
폴더 내 기존 `handoff_*.md` 있으면 마지막 파일을 읽어 **체인 연장** 형태로 작성. 이전 세션 내용을 복사하지 말고 "이전 세션: handoff_YYYYMMDD_prev.md" 링크만.

### (5) 사용자에게 생성 경로 보고

### (6) compact 수행 여부 확인 + 프롬프트 제공
핸드오프 메모 작성·보고 직후, **`AskUserQuestion`** 으로 compaction 수행 여부를 확인한다.

- 옵션: "compact 수행" (세션 종료 맥락이면 Recommended + 첫 옵션) / "미수행".
- "compact 수행" 선택 시, 아래 §"compact 프롬프트 구성" 기준으로 작성한 compact 프롬프트를 코드 블록으로 제공한다.
- compaction 실행 자체는 사용자가 `/compact` 로 트리거한다 — 스킬은 프롬프트 제공까지만.

## compact 프롬프트 구성

compact 프롬프트는 방금 작성한 핸드오프 메모를 **단일 소스**로 가리키고 핵심만 요약한다. 메모 전문을 복제하지 않는다. 필수 요소:

1. **핸드오프 메모 경로** — `@.Codex/handoffs/handoff_YYYYMMDD_<topic>.md` 를 단일 소스로 명시. "§3 우선순위·§6 착수 가이드 보존 필수" 지시 포함.
2. **현재 상태** — 본 세션 코드/문서 변경 1~2줄 요약.
3. **다음 착수 예정** — 다음 세션이 시작할 작업 (메모 §3 1순위와 일치).
4. **유지할 사용자 성향** — AGENTS.md 참조로 (규범 본문 복사 금지). 이번 세션 강화 성향만 간단히.
5. **미결 검토 항목** — 승인 대기·후속 검토 항목.
