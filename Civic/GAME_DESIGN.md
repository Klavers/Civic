# Civic 게임 기본 기획서

## 1. 게임 개요

`Civic`은 문명의 발전을 주제로 한 증분형 게임이다.

플레이어는 인구를 기반으로 건물을 짓고, 건물이 생산하는 자원으로 GDP를 성장시키며, 축적된 과학으로 기술을 연구해 다음 시대와 더 복잡한 생산 체인을 개방한다.

핵심 재미는 단순 클릭이 아니라 다음 흐름을 반복하며 경제 병목을 찾아 해소하는 데 있다.

1. 인구 확보
2. 건물 건설
3. 자원 생산
4. GDP 상승
5. 국고와 건설력 확보
6. 더 많은 건물 건설
7. 과학 축적
8. 기술 연구와 시대 발전

클릭 요소는 주력 시스템이 아니다. 특정 건물을 클릭해 세부 정보를 보거나 일부 자원/연구를 보조적으로 부스팅하는 정도로만 고려한다.

## 2. 시대와 기술

시대는 다음 순서로 구성한다.

1. 원시시대
2. 고대시대
3. 고전시대
4. 중세시대
5. 르네상스
6. 산업시대
7. 현대

시대 진입은 별도 조건 없이 기술로만 결정한다.

- 다음 시대에 해당하는 기술 중 아무거나 하나를 연구하면 즉시 다음 시대로 진입한다.
- 이전 시대 기술은 자동으로 개방되지 않는다.
- 지나간 시대의 기술도 이후에 계속 연구할 수 있다.
- 기술 연구에는 과학을 소비한다.
- 필요한 과학 포인트를 보유하고 있으면 기술을 즉시 개방할 수 있고, 개방 시 과학 잔액이 차감된다.

시대별 건물 목록, 기술 목록, 건물 계수는 추후 데이터로 입력한다.

## 3. 주요 수치 자원

### 인구

인구는 실제 사람 수가 아니라 노동 단위이자 건물 수용력이다.

- 건물 1개는 인구 1을 필요로 한다.
- 이 규칙은 고정이며 변경하지 않는다.
- 전체 건물 수는 현재 인구를 초과해서 지을 수 없다.
- 단, `population`을 생산하는 집 계열 건물은 인구 한도를 늘리는 건물이므로 현재 건물 수가 인구와 같아도 건설할 수 있다.
- 이미 지어진 건물 수가 인구보다 많아지는 상황이 생겨도 건물은 비활성화되지 않는다.
- 이 경우 UI에는 `건물 수 / 인구` 형태로 표시하고, 추가 건설만 막는다.
- 플레이어가 다른 건물을 짓고 싶다면 기존 건물을 철거해야 한다.

예시:

- 건물 50개, 인구 48이면 `50 / 48`로 표시한다.
- 이 상태에서 기존 건물은 계속 작동한다.
- 새 건물은 지을 수 없다.

### 과학

과학은 기술 연구에 쓰이는 누적 자원이다.

- 기본적으로 인구 1당 과학 +1/s를 생산한다.
- 기술 연구 시 필요한 과학을 지불하고 차감한다.
- 과학은 GDP에 포함하지 않는다.

### 국고

국고는 건설부문 운영비를 지불하기 위한 재정 자원이다.

국고 증가량은 다음 값의 합으로 계산한다.

```text
국고 증가량 = GDP × 세금비율 + 수도 기본 국고 생산량
```

- 세금비율은 기술 발전에 따라 점점 증가한다.
- 수도가 생산하는 식량계 자원이나 목재 등은 GDP에 포함되고, 그 GDP를 통해 국고 증가에 기여한다.
- 수도는 별도의 기본 국고도 소량 생산한다.
- 국고는 음수가 될 수 없다.

### 건설력

건설력은 건물을 짓기 위해 축적되는 자원이다.

- UI에는 `보유량 (+초당 증가량)` 형태로 표시한다.
- 건물은 필요한 건설력을 모두 보유하고 있으면 즉시 지어진다.
- 건설에는 별도 대기 시간이나 공사 시간이 없다.
- 건물 건설 비용은 건설력만 사용한다.
- 건설력은 건설부문을 통해 생산한다.

## 4. 요소 자원과 식량

### 요소 자원

요소 자원은 게임 내 생산/소비/가격/GDP 계산에 직접 참여하는 자원이다.

초기 예시는 다음과 같다.

- 1차 자원: 나무, 밀
- 2차 자원: 공구, 종이

1차 자원은 다른 자원을 소비하지 않고 건물에서 바로 생산된다.
2차 자원은 다른 자원을 소비하는 건물을 통해 생산된다.

### 식량

식량은 개별 자원이 아니라 여러 식량계 자원의 통산 값이다.

예시 식량계 자원:

- 밀
- 고기
- 식료품

식량 총량은 다음과 같이 계산한다.

```text
식량 보유량 = Σ(식량계 자원 보유량 × 식량 환산비)
식량 증가량 = Σ(식량계 자원 증가량 × 식량 환산비)
```

식량 UI는 다음 계층 구조로 표시한다.

```text
식량 아이콘 = 모든 식량계 자원 보유량 × 환산비의 합 (+환산비 적용 증가량)
  - 밀 = 밀 보유량 (+밀 증가량)
  - 고기 = 고기 보유량 (+고기 증가량)
  - 식료품 = 식료품 보유량 (+식료품 증가량)
```

식량 자체에는 가격이 없다.
식량 자체는 GDP에 포함하지 않는다.

대신 밀, 고기, 식료품 같은 하위 식량계 자원은 각각 가격을 가지고 GDP에 포함된다.

## 5. 건물

### 수도

수도는 게임 시작 시 보유하는 기본 건물이다.

- 플레이어가 직접 업그레이드하지 않는다.
- 복잡한 관리 기능을 갖지 않는다.
- P01 기준 수도는 밀, 목재, 국고, 건설력을 각각 +1/s 생산한다.
- 클릭하면 현재 문명의 세부 개괄 정보를 보여주는 용도로 사용한다.

### 생산건물

생산건물은 인구 1을 점유하고 자원을 생산한다.

초기 예시:

- 벌목장: 나무 생산
- 밀농장: 밀 생산
- 공구공장: 나무 소비, 공구 생산
- 제지소: 나무 소비, 종이 생산

일부 건물은 아무 투입물 없이 산출물을 만든다.
일부 건물은 다른 자원을 소비해 더 높은 가치의 자원을 만든다.

### 수치자원 생산 건물

수치자원을 생산하는 건물도 생산건물과 같은 규칙을 따른다.

초기 예시:

- 건설부문: 건설력 생산
- 오두막/집: 인구 생산

### 건설부문

건설부문은 건설력을 생산하는 건물이다.

- 건설부문 1개는 인구 1을 점유한다.
- 건설부문은 필요한 건설자재를 계속 소비한다.
- 건설자재 소비 비용은 국고에서 지불한다.
- 자재 공급이 부족하면 다른 생산건물과 같은 효율 규칙으로 건설력 생산량이 감소한다.
- 국고가 부족하면 지불 가능한 국고만큼만 작동한다.
- 국고는 0 아래로 내려가지 않는다.

건설부문 확장 시에는 예상 추가 운영비를 표시한다.

```text
예상 추가 운영비 = 필요 건설자재 수량 × 현재 가격
```

### 집 계열 건물

집 계열 건물은 인구를 즉시 증가시키는 건물이다.

- 기본 집은 인구 +2를 생산한다.
- 식량과 목재를 충분히 공급받으면 +2가 적용된다.
- 식량 또는 목재 중 하나만 공급되면 +1이 적용된다.
- 둘 다 공급되지 않으면 +0이 적용된다.
- 기술 발전에 따라 집의 기본 인구 증가량은 +2 이상으로 확장될 수 있다.

집도 다른 생산건물과 같은 투입 자원 충족률 규칙을 사용한다.

## 6. 생산 효율과 품귀

자원이 부족할 때는 소비 우선순위를 두지 않는다.

- 같은 자원을 요구하는 모든 건물이 동시에 소비한다.
- 공급이 부족하면 해당 자원을 요구하는 모든 건물의 효율이 같은 비율로 감소한다.
- 자원별 공급률은 `실제 공급량 / 정상 가동에 필요한 수요량`으로 계산한다.
- 여러 투입 자원이 필요한 건물은 각 투입 자원 종류별 공급률의 단순 평균을 생산 효율로 사용한다.
- 투입 자원의 요구 수량은 효율 평균의 가중치로 사용하지 않는다.

예시 1:

```text
건물 10개가 자원 A를 총 10개 필요로 한다.
자원 A가 9개만 공급된다.

자원 A 공급률 = 90%
건물 생산 효율 = 90%
```

예시 2:

```text
건물 10개가 자원 A와 자원 B를 소비한다.
자원 A는 100% 공급된다.
자원 B는 50% 공급된다.

생산 효율 = (100% + 50%) / 2 = 75%
```

예시 3:

```text
건물이 나무 2개와 철 1개를 요구한다.
나무가 부족하고 철은 충분하다.

나무 공급률 = 0%
철 공급률 = 100%
생산 효율 = (0% + 100%) / 2 = 50%
```

자원 보유량 상한은 없다.

## 7. 가격과 GDP

### 가격

모든 요소 자원은 수요와 공급에 따라 변동 가격을 가진다.

- 공급이 넘치면 가격이 떨어진다.
- 공급이 부족하면 가격이 오른다.
- 공급과 수요가 일치할 때 기준 가격이 된다.
- 가격 변화는 선형 보간을 사용한다.

가격 계산 기준:

- 수요: 모든 건물이 정상 가동하려면 필요한 양
- 공급: 실제 생산량

수요/공급 비율에 따른 가격 범위:

```text
공급 / 수요 <= 0.25  => 가격 = 기준가 × 200%
공급 / 수요 = 1.0    => 가격 = 기준가 × 100%
공급 / 수요 >= 4.0   => 가격 = 기준가 × 12.5%
```

중간 구간은 선형으로 보간한다.

- `0.25 ~ 1.0` 구간: 200%에서 100%까지 선형 하락
- `1.0 ~ 4.0` 구간: 100%에서 12.5%까지 선형 하락

수요가 0인 자원은 최저가인 기준가의 12.5%로 계산한다.

### GDP

GDP는 실제 생산량과 현재 변동 가격의 곱을 모두 더해 계산한다.

```text
GDP = Σ(자원별 실제 생산량 × 현재 변동 가격)
```

GDP에 포함되는 것:

- 나무
- 밀
- 고기
- 식료품
- 공구
- 종이
- 기타 모든 요소 자원

GDP에 포함하지 않는 것:

- 식량 통산값
- 과학
- 국고
- 건설력

식량계 하위 자원은 각각 GDP에 포함되지만, 식량 통산값은 중복 계산을 막기 위해 GDP에 포함하지 않는다.

## 8. 초기 상태

게임 시작 시 기본 상태는 다음과 같다.

- 수도 1개
- 인구 3
- 식량 5
- 건설력 5
- 과학 생산량: 인구 1당 +1/s

초기 해금 건물과 기술 목록은 추후 데이터로 확정한다.

## 9. UI 기본 방향

### 자원 패널

화면 우측에는 다양한 자원을 표시한다.

기본 형식:

```text
아이콘  보유량  (+초당 증가량)
```

초당 증가량은 음수도 표시한다.

식량은 상위 자원으로 표시하고, 펼쳤을 때 하위 식량계 자원을 보여준다.

```text
식량  20  (+3/s)
  밀  10  (+1/s)
  고기  5  (+0.5/s)
```

좌측 자원 상세 패널은 우측 요약 패널과 역할을 분리한다. 자원 상세에는 +1 건물 예상 변화치를 표시하지 않고, 자원별 생산처와 생산처별 GDP 기여를 표시한다.

```text
나무 | 생산 6/s | 가격 x0.13 | GDP 1.56
  수도 x1 | 1/s | GDP 0.26
  벌목장 x5 | 5/s | GDP 1.30
```

GDP에서 제외되는 자원은 GDP 칸을 `GDP 제외` 또는 `-`로 표시한다.

### 건물 패널

건물 패널은 건설 선택용 표로 표시한다.

```text
건물명 | 개수 | 건설비용 | 투입/산출 | GDP 변화 | 건설
```

- +1 건물 예상 변화치는 자원 상세이 아니라 건물 패널의 `투입/산출`과 `GDP 변화` 열에 표시한다.
- 투입/산출은 고정 높이 한 줄에 최대 일부 항목만 표시하고, 초과 항목은 `+N`으로 요약한다.
- `+N`에 마우스를 올리면 전체 투입/산출 목록을 툴팁으로 표시한다.
- 건설 불가능한 건설 버튼은 어두운 색으로 비활성화하고, 마우스를 올리면 건설 불가능 사유를 툴팁으로 표시한다.
- 패널 콘텐츠가 패널보다 크면 ScrollRect로 스크롤한다.

### 건물 수 표시

건물 수와 인구는 다음과 같이 표시한다.

```text
건물 50 / 인구 48
```

건물 수가 인구 이상이면 추가 건설 버튼을 비활성화하거나 경고를 표시한다. 단, `population`을 생산하는 집 계열 건물은 이 제한의 예외로 표시한다.

### 건설부문 표시

건설부문 확장 UI에는 다음 정보를 보여준다.

- 현재 건설력 보유량
- 초당 건설력 증가량
- 추가 건설부문이 요구하는 자재
- 추가 운영비 예상값
- 국고 부족 시 예상 가동률

### 가격과 품귀 표시

자원 UI에는 필요 시 다음 정보를 표시한다.

- 현재 가격
- 기준가 대비 가격 배율
- 공급률
- 품귀 여부
- 이 자원을 요구하는 건물들의 평균 효율

## 10. 보류 항목

다음 항목은 추후 데이터 입력 단계에서 확정한다.

- 시대별 기술 목록
- 시대별 건물 목록
- 건물별 생산량
- 건물별 소비 자원
- 자원별 기준 가격
- 자원별 식량 환산비
- 세금비율 증가 테크
- 건설부문 자재 요구량
- 집 계열 건물의 시대별 인구 증가량
